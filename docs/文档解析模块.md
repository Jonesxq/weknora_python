# 文档解析模块

## 目标

`app.docreader` 在当前 Python 进程内把上传文件转换成 Markdown、图片引用和元数据，不使用 gRPC 或其他网络协议。

当前第一阶段支持：

- PDF
- DOCX
- Markdown
- XLS / XLSX
- JPG、JPEG、PNG、GIF、BMP、TIFF、WebP

暂未启用旧 DOC、PPT/PPTX、网页、EPUB、MHTML 和 OpenDataLoader。这些格式需要额外的系统程序或较重依赖，应按实际需求单独接入。

## 调用方式

```python
from app.api.dependencies import get_document_reader


reader = get_document_reader()
result = reader.parse_bytes(
    file_name="example.pdf",
    content=file_bytes,
)

markdown = result.content
images = result.images
metadata = result.metadata
```

`DocumentReaderService` 是项目其他模块应依赖的稳定入口。RAG、API 和后台任务不应直接导入具体的 PDF、DOCX 或 Excel 解析器。

## FastAPI 接口

文档解析已经注册为 `POST /api/v1/documents/parse`。请求类型为
`multipart/form-data`，文件字段名必须是 `file`。

默认最大上传文件为 50 MiB，可以通过环境变量
`DOCREADER_MAX_FILE_SIZE_MB` 调整。接口主要状态码如下：

- `200`：解析成功。
- `413`：文件超过大小限制。
- `415`：文件扩展名不受支持。
- `422`：文件名缺失、文件为空或文档内容损坏。

从项目根目录启动开发服务：

```powershell
# 启动 FastAPI 开发服务；代码变更后自动重载
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

服务启动后，可以打开 <http://127.0.0.1:8000/docs>，在 Swagger 页面中选择
`POST /api/v1/documents/parse`，点击“Try it out”并上传文件。

也可以在另一个 PowerShell 窗口执行：

```powershell
# 检查服务是否已经启动
Invoke-RestMethod http://127.0.0.1:8000/health

# 上传并解析文档；请把示例路径换成你本机的真实文件路径
curl.exe -X POST "http://127.0.0.1:8000/api/v1/documents/parse" -F "file=@E:\path\example.pdf"
```

成功响应包含 `file_name`、`content_type`、`size_bytes`、`content`、
`images` 和 `metadata`。其中 `content` 是解析得到的 Markdown 文本。

## 处理边界

```text
上传文件
  → DocumentReaderService
  → 格式解析器
  → Markdown、图片和元数据
  → app/rag/ingestion 负责 OCR、切块、向量化和入库
```

扫描 PDF 会被渲染为图片，但当前模块不执行 OCR。调用方应根据 `metadata["scanned_page_count"]` 或图片结果进入后续 OCR 流程。

解析器是同步代码。异步 Web 接口应把解析调用放入线程池；大文件和批量文件应交给 `app/workers`，避免阻塞请求线程。
